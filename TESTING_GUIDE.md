# FocusGuard 试用期和升级功能测试指南

## 快速测试步骤

### 1. 测试试用期（剩余 7 天）

```bash
# 设置为新用户状态
npm run test:license new-user

# 启动应用
npm start
```

**预期结果：**
- ✅ 顶部横幅显示："试用期：剩余 7 天"（蓝色背景）
- ✅ 显示"升级 Pro"按钮
- ✅ Pro 功能可用（因为在试用期内）
- ✅ 随机拍照、解锁答题功能正常工作

### 2. 测试试用期即将到期（剩余 1 天）

```bash
# 设置为试用期第 6 天
npm run test:license trial-6-days

# 启动应用
npm start
```

**预期结果：**
- ✅ 顶部横幅显示："试用期：剩余 1 天"（蓝色背景）
- ✅ 显示"升级 Pro"按钮
- ✅ Pro 功能仍然可用

### 3. 测试试用期已过期

```bash
# 设置为试用期过期状态
npm run test:license trial-expired

# 启动应用
npm start
```

**预期结果：**
- ✅ 顶部横幅显示："试用已到期，升级 Pro 解锁全部功能"（红色背景）
- ✅ 显示"升级 Pro"按钮
- ✅ Pro 功能被锁定：
  - 随机拍照选项变灰且不可点击
  - 解锁答题选项变灰且不可点击
  - 点击"查看详细统计图表"显示 Pro 锁定弹窗
  - 点击"导出 PDF 周报"显示 Pro 锁定弹窗

### 4. 测试购买流程入口

在试用期过期状态下，测试以下按钮是否都能打开购买页面：

#### 测试点 1：顶部横幅按钮
1. 点击顶部横幅的"升级 Pro"按钮
2. **预期：** 在浏览器中打开购买页面

#### 测试点 2：Pro 锁定弹窗
1. 点击"查看详细统计图表"或"导出 PDF 周报"
2. 弹出 Pro 锁定提示窗口
3. 点击"获取 FocusGuard Pro"按钮
4. **预期：** 在浏览器中打开购买页面

#### 测试点 3：设置页面购买链接
1. 切换到"设置" Tab
2. 滚动到"许可证"区域
3. 点击"还没有许可证？立即购买 →"链接
4. **预期：** 在浏览器中打开购买页面

### 5. 测试许可证激活

#### 准备工作
在 LemonSqueezy 后台创建测试许可证，或使用测试模式购买。

#### 激活步骤
1. 在"设置" Tab 的许可证区域
2. 在输入框中输入许可证密钥（例如：`XXXX-XXXX-XXXX-XXXX`）
3. 点击"激活"按钮

**预期结果（成功）：**
- ✅ 显示"激活成功！"（绿色文字）
- ✅ 顶部横幅变为："Pro 已激活"（绿色背景）
- ✅ "升级 Pro"按钮消失
- ✅ 许可证区域显示密钥前缀（例如：`XXXX-XXX...`）
- ✅ 显示"停用"按钮
- ✅ Pro 功能解锁：
  - 随机拍照选项可用
  - 解锁答题选项可用
  - 可以查看详细统计图表
  - 可以导出 PDF 周报

**预期结果（失败 - 无效密钥）：**
- ✅ 显示错误："密钥无效，请检查输入是否正确（注意大小写和连字符）"（红色文字）

**预期结果（失败 - 网络错误）：**
- ✅ 显示错误："网络连接失败：...。请检查网络后重试，如持续失败请联系客服。"（红色文字）

### 6. 测试许可证持久化

1. 激活许可证（参考步骤 5）
2. 关闭应用
3. 重新启动应用

**预期结果：**
- ✅ 自动保持 Pro 激活状态
- ✅ 顶部横幅显示："Pro 已激活"
- ✅ Pro 功能可用

### 7. 测试许可证停用

1. 在已激活状态下
2. 在"设置" Tab 的许可证区域
3. 点击"停用"按钮

**预期结果：**
- ✅ 显示"已停用"（绿色文字）
- ✅ 如果仍在试用期内：
  - 顶部横幅显示："试用期：剩余 X 天"
  - Pro 功能仍可用
- ✅ 如果试用期已过：
  - 顶部横幅显示："试用已到期，升级 Pro 解锁全部功能"
  - Pro 功能被锁定

### 8. 测试离线缓存


1. 确认 Pro 状态正常

**预期结果：**
- ✅ 仍然显示 Pro 激活状态（使用 7 天缓存）
- ✅ Pro 功能可用

### 9. 测试缓存过期

```bash

# 断开网络
# 启动应用
```

**预期结果：**

## 常见问题排查

### 问题：点击"升级 Pro"没有反应

**排查步骤：**
1. 检查 `.env` 文件是否存在且配置正确
2. 检查 `PURCHASE_URL` 是否是完整的 URL
3. 打开开发者工具（Cmd+Option+I）查看控制台错误
4. 手动在浏览器中访问购买 URL 确认可用

### 问题：激活许可证失败

**排查步骤：**
1. 检查网络连接
2. 确认密钥格式正确（无多余空格）
3. 在 LemonSqueezy 后台确认密钥有效
4. 查看错误提示的具体内容
5. 打开开发者工具查看网络请求

### 问题：Pro 功能没有解锁

**排查步骤：**
1. 确认顶部横幅显示"Pro 已激活"
2. 检查许可证文件内容：
   ```bash
   # macOS
   cat ~/Library/Application\ Support/lockscreen-focus/license.json
   ```
3. 确认 `status` 为 `"active"`
4. 确认 `lastValidationResult` 为 `true`
5. 重启应用

### 问题：试用期不正确

**排查步骤：**
1. 检查许可证文件中的 `firstLaunchAt` 时间戳
2. 使用测试脚本重置状态：
   ```bash
   npm run test:license new-user
   ```
3. 或手动删除许可证文件

## 自动化测试建议

如果需要更完整的自动化测试，可以考虑：

1. **单元测试**：测试 `license.js` 中的各个函数
2. **集成测试**：使用 Spectron 测试 Electron 应用
3. **E2E 测试**：使用 Playwright 或 Puppeteer 测试完整流程

示例单元测试（使用 Jest）：

```javascript
const license = require('./license');

describe('License Module', () => {
  test('isTrialActive returns true for new user', () => {
    // 模拟新用户数据
    const result = license.isTrialActive();
    expect(result).toBe(true);
  });

  test('getTrialDaysLeft returns 7 for new user', () => {
    const days = license.getTrialDaysLeft();
    expect(days).toBe(7);
  });
});
```
